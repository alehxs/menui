## Directory Purposes

**`backend/app/`:**
- Purpose: Python FastAPI application
- Contains: API endpoints, business logic, configuration
- Key files:
  - `backend/app/main.py`: FastAPI app, endpoints, request handlers
  - `backend/app/config.py`: Environment-based configuration
  - `backend/app/__init__.py`: Package marker

**`backend/`:**
- Purpose: Backend service root
- Contains: Application code, dependencies, environment config
- Key files:
  - `backend/requirements.txt`: Python package dependencies
  - `backend/.env`: API keys and secrets (not committed)
  - `backend/venv/`: Virtual environment (not committed)

**`Menui/Services/`:**
- Purpose: iOS business logic layer
- Contains: Service classes for OCR, parsing, API calls, camera
- Key files:
  - `Menui/Services/OCRService.swift`: Apple Vision text recognition
  - `Menui/Services/DishParserService.swift`: OCR output filtering
  - `Menui/Services/APIService.swift`: Backend HTTP client
  - `Menui/Services/CameraManager.swift`: AVFoundation camera wrapper

**`Menui/Views/`:**
- Purpose: SwiftUI view components
- Contains: UI screens and reusable components
- Key files:
  - `Menui/Views/CameraView.swift`: Camera capture interface
  - `Menui/Views/CameraPreview.swift`: UIViewRepresentable camera preview
  - `Menui/Views/ResultsView.swift`: Dish list with images

**`Menui/Assets.xcassets/`:**
- Purpose: iOS app resources
- Contains: App icon, images, color sets
- Key subdirectories:
  - `AppIcon.appiconset`: App icon variants
  - `AccentColor.colorset`: Global accent color
  - Sample menu images for testing

**`Menui/`:**
- Purpose: iOS app root source directory
- Contains: Entry point, root navigation, feature modules
- Key files:
  - `Menui/MenuiApp.swift`: @main entry point
  - `Menui/MainTabView.swift`: Tab-based navigation root
  - `Menui/Info.plist`: Camera permissions, app metadata

**`.planning/codebase/`:**
- Purpose: GSD codebase documentation
- Contains: Architecture, structure, stack analysis
- Generated by: GSD mapping agents

## Naming Conventions

**Swift Files:**
- PascalCase with descriptive suffixes
  - Views: `CameraView.swift`, `ResultsView.swift`
  - Services: `OCRService.swift`, `APIService.swift`
  - Managers: `CameraManager.swift`
  - App/Navigation: `MenuiApp.swift`, `MainTabView.swift`

**Python Files:**
- snake_case for modules
  - `main.py`: Primary application file
  - `config.py`: Configuration module

**Directories:**
- PascalCase for iOS (Swift convention): `Services/`, `Views/`
- lowercase for backend (Python convention): `backend/`, `app/`

**Classes:**
- Swift: PascalCase (`OCRService`, `CameraManager`, `APIService`)
- Python: PascalCase for Pydantic models (`DishRequest`, `DishImagesResponse`)

**Functions:**
- Swift: camelCase (`recognizeText`, `fetchDishImages`)
- Python: snake_case (`fetch_images_from_google`, `get_cached_images`)

## Where to Add New Code

**New iOS View:**
- Primary code: `Menui/Views/[ViewName].swift`
- Navigation: Update `Menui/MainTabView.swift` if adding tab
- Supporting UI: Co-locate in `Views/` directory

**New iOS Service:**
- Implementation: `Menui/Services/[ServiceName].swift`
- Pattern: Class with methods, optionally ObservableObject if state needed
- Usage: Instantiate in views or use singleton pattern like APIService

**New iOS Model:**
- Models directory does not exist yet
- Recommendation: Create `Menui/Models/` for data structures
- Currently: Models defined inline in `APIService.swift`

**New Backend Endpoint:**
- Implementation: Add route decorator and function in `backend/app/main.py`
- Pattern: Follow existing structure (Pydantic models, async function, error handling)
- Location: Group with related endpoints (API section marked with comments)

**New Backend Configuration:**
- Add to: `backend/app/config.py`
- Pattern: Load from `os.getenv()` with optional default
- Usage: Import config module in main.py

**New Backend Feature/Service:**
- Current pattern: Functions in `backend/app/main.py`
- Recommendation for growth: Create `backend/app/services/` directory
- File structure not yet modularized (all code in main.py)

**Assets (Images, Icons):**
- Location: `Menui/Assets.xcassets/`
- Pattern: Create `.imageset` or `.colorset` subdirectory
- Xcode manages asset catalog structure
